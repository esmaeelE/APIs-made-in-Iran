# راهنمای مشارکت

از اینکه قصد مشارکت در این پروژه را دارید سپاسگزاریم! با کمک شما این لیست جامع‌تر و بهتر می‌شود.

## راه‌های مشارکت

### ۱. گزارش API جدید
اگر سایت یا سرویس ایرانی می‌شناسید که API ارائه می‌کند، لطفاً یک Issue جدید ایجاد کنید و اطلاعات زیر را در آن قرار دهید:

- نام سرویس
- آدرس وب‌سایت
- توضیحات مختصر درباره خدمات
- دسته‌بندی مناسب (در صورت اطلاع)
- وضعیت قیمت (رایگان، پولی، یا هر دو)

### ۲. ارسال Pull Request

#### اضافه کردن API جدید
۱. مخزن را Fork کنید
۲. یک branch جدید ایجاد کنید
۳. API جدید را به انتهای دسته‌بندی مناسب اضافه کنید
۴. اگر دسته‌بندی مناسبی وجود ندارد، به بخش «نیازمند دسته‌بندی» اضافه کنید
۵. Pull Request ارسال کنید

#### رفع مشکلات
- اصلاح لینک‌های شکسته
- به‌روزرسانی اطلاعات نادرست
- بهبود توضیحات

#### نکات مهم فرمت

**وضعیت قیمت:**
| نماد | معنی |
|---|---|
| :free: | کاملاً رایگان |
| :moneybag: | کاملاً پولی |
| :free::moneybag: | رایگان و پولی |
| :question: | نامشخص |

**وضعیت دسترسی:**
| نماد | معنی |
|---|---|
| :iran: | فقط داخل اینترانت ایران (ایران‌اکسس) |
| :globe_with_meridians: | در اینترنت قابل دسترس |

> [!NOTE]
> برای سرویس‌هایی که فقط از داخل ایران قابل دسترس هستند، از فرمت `<!-- markdown-link-check-disable-line -->` در انتهای خط استفاده کنید تا لینک‌ها توسط GitHub Action به اشتباه به عنوان شکسته شناسایی نشوند.

### ۳. بررسی لینک‌ها
لینک‌ها پس از هر commit توسط GitHub Action بررسی می‌شوند. اگر لینک شکسته‌ای پیدا کردید، لطفاً Issue ایجاد کنید.

## قوانین

۱. لطفاً قبل از ارسال API جدید، مطمئن شوید که در لیست وجود ندارد
۲. توضیحات باید دقیق و مختصر باشد
۳. لینک‌ها باید معتبر و قابل دسترس باشند
۴. از دسته‌بندی صحیح اطمینان حاصل کنید
